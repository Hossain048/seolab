What is Big-O Notation?

Big-O notation describes the worst-case time or space complexity of an algorithm as the input size n grows. It tells you how fast the runtime increases — not the exact time, but the growth rate.

Think of it this way: if you double the input, does the runtime double? Square? Stay the same? Big-O answers that question.

The Most Common Complexities

O(1) — Constant Time

The operation takes the same time regardless of input size. Accessing an array element by index is O(1).

def get_first(arr):
    return arr[0]  # Always one operation, O(1)

O(log n) — Logarithmic Time

Halves the problem each step. Binary search is the classic example.

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target: return mid
        elif arr[mid] < target: left = mid + 1
        else: right = mid - 1
    return -1

O(n) — Linear Time

Time grows directly with input. A simple loop through an array is O(n).

def find_max(arr):
    max_val = arr[0]
    for x in arr:        # n iterations
        if x > max_val:
            max_val = x
    return max_val

O(n²) — Quadratic Time

A loop inside a loop. Bubble sort is O(n²) — avoid for large inputs.

def bubble_sort(arr):
    n = len(arr)
    for i in range(n):          # O(n)
        for j in range(n-i-1): # O(n)
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]

Quick Reference Table

  • O(1) — Hash table lookup, array access
  • O(log n) — Binary search, balanced BST operations
  • O(n) — Linear search, single loop
  • O(n log n) — Merge sort, heap sort
  • O(n²) — Bubble sort, nested loops
  • O(2ⁿ) — Recursive Fibonacci (naive)

Why Does Big-O Matter for CSE Students?

Every technical interview at FAANG and top companies tests algorithmic thinking. Understanding Big-O lets you:

  • Choose the right data structure for a problem
  • Predict performance before writing code
  • Optimize existing solutions systematically
  • Communicate trade-offs clearly in interviews
